# Rejected and Deliberately Deferred Ideas

**Status:** Approved record

| Idea | Status | Reason |
|---|---|---|
| Individual tree simulation | Rejected | Aggregate forest coverage creates the intended choices at sustainable cost. |
| Individual fish/ecosystem simulation | Rejected | Resolve fishing from local state and action-time inputs. |
| Accelerated world calendar | Rejected baseline | Use real calendar time with compressed action durations. |
| Offline hunger death | Rejected | Punishes absence rather than decisions. |
| Unlimited offline personal work queue | Rejected | Creates AFK farming; only started actions complete. |
| Manual bookkeeping, VAT and tax filing | Rejected | Administrative work is automatic. |
| Player-written executable in-world software | Rejected | Offer official services and an external API instead. |
| Omniscient remote plot inspection | Rejected | Information must have an in-world source. |
| Random independent resources per plot | Rejected | Resources follow coherent geography and geology. |
| Selecting building quality from a menu | Rejected | Quality emerges from design, inputs and labour. |
| Real-money purchase of world currency | Rejected | Breaks economic integrity. |
| Multiple active characters per account | Rejected baseline | Enables self-dealing and weakens identity. |
| Full childhood simulation | Rejected | New NPCs enter as adults; education starts at vocational level. |
| Microservices at project start | Deferred/rejected baseline | Modular monolith gives safer transactions and lower complexity. |
| War in MVP | Deferred | Requires mature politics, logistics, property and safety systems. |
| Full medical simulator | Deferred | Medium-depth health provides the desired consequences. |

Reconsideration requires an explicit owner decision and an update explaining what changed.
