# Architectural Decision Records

Use ADRs for consequential choices that are expensive to reverse.

## Format

```markdown
# ADR-NNN: Title

**Status:** Proposed | Accepted | Superseded
**Date:** YYYY-MM-DD

## Context
## Decision
## Alternatives
## Consequences
## Revisit when
```

## Required before implementation

1. modular monolith and domain layout
2. plot scale and spatial representation
3. persistent action completion and idempotency
4. ledger and economic conservation
5. event outbox and queue retry strategy
6. identifiers and audit model
7. initial MVP production chain
